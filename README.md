# Bungalooo 🏖️

Уеб приложение за **сравнение и избор на бунгала по Южното българско Черноморие**.
Няколко души (Ник, Дани, Мари, Иван) разглеждат предложения, добавят бележки, сменят статуси
и виждат актуалната информация от всяко устройство.

- 🌐 **Живо приложение:** <https://kartalovnik.github.io/bungalooo/>
- 💻 **Код (GitHub):** <https://github.com/KartalovNik/bungalooo>

**Всичко е само на GitHub — без други услуги:**
- **Хостинг → GitHub Pages** (публичен линк, инсталира се на телефона, безплатно).
- **Данни → файл `data.json` в клон `data` на същото хранилище.** Четенето е публично за
  всички; записът минава през GitHub с общ **код за редактиране** (виж по-долу).
  Промените се появяват при другите за секунди (при редакторите) до няколко минути (при
  публичните посетители).

> Бонус: понеже всяка промяна е commit в клона `data`, имате **пълна история** на всичко —
> кой и кога какво е пипал, с възможност за връщане назад.

## 1. Как да разглеждам и ползвам

Отворете <https://kartalovnik.github.io/bungalooo/> на телефон или компютър.

- **Публичен режим** (по подразбиране) — всеки разглежда, търси, филтрира, отваря линкове.
- **Режим за редактиране** — натиснете **Вход**, въведете общия **код за редактиране**, изберете
  кой сте (Ник / Дани / Мари / Иван). Всяка ваша промяна се записва с вашето име и цвят.

**Добавяне на приложението на телефона:** отворете сайта → менюто на браузъра → „Добави на
началния екран".

---

## 2. Код за редактиране

„Кодът за редактиране" е **личен код за достъп на GitHub (token)**. Той работи като обща парола:
създава се **веднъж**, споделя се между четиримата, и всеки го въвежда веднъж на своето устройство.
Кодът се пази само в браузъра — **никога не стои в кода на сайта.**

### Създаване (прави се веднъж, от Ник):

1. Влезте в GitHub → <https://github.com/settings/tokens?type=beta> (Settings → Developer
   settings → **Fine-grained tokens**) → **Generate new token**.
2. **Token name:** `Bungalooo`
3. **Expiration:** изберете дълъг срок (напр. 1 година; отбележете си кога да го подновите).
4. **Resource owner:** `KartalovNik`.
5. **Repository access:** **Only select repositories** → изберете **`bungalooo`**.
6. **Permissions → Repository permissions → Contents:** сложете **Read and write**.
7. **Generate token** → копирайте кода (започва с `github_pat_…`).

Това е **кодът за редактиране**. Споделете го с Дани, Мари и Иван (напр. в семейния чат).
Всеки го въвежда веднъж през бутона **Вход** в приложението.

> Ако кодът изтече или се компрометира: изтрийте го от същата страница и създайте нов.
> Данните не се губят — те са в клона `data`.

## 3. Публикуване на промени по кода

(Само когато променяте **самото приложение** — не за ежедневните данни, те се пазят автоматично.)

```bash
npm install      # само първия път
npm run deploy
```

Прави build и качва новата версия на GitHub Pages. След минута-две е живо на
<https://kartalovnik.github.io/bungalooo/>.

## 5. Настройки за поддръжка

Всичко важно е в **`src/config.js`** (след промяна → `npm run deploy`):

- **Хора (имена и цветове):**

  ```js
  export const USERS = [
    { id: 'nik',  name: 'Ник',  color: '#2563eb' },
    { id: 'dani', name: 'Дани', color: '#0d9488' },
    { id: 'mari', name: 'Мари', color: '#db2777' },
    { id: 'ivan', name: 'Иван', color: '#d97706' },
  ]
  ```

- **Статуси, удобства, предложени райони** — също там.

Ако преименувате GitHub хранилището, сменете `REPO` в **`vite.config.js`** и стойностите в
**`src/github.js`**.

## 6. Резервно копие

- **В приложението:** меню **⋯ → „Изнеси JSON (резервно копие)"** (и „Внеси / възстанови").
  Поддържа се и CSV.

  който може да се разгледа и върне.

## 8. Функции

- Карти: снимка, име, място, цена, разстояние до плажа, рейтинг, капацитет, удобства с икони,
  статус, любимо, кратка бележка.
- **Статуси** с цвят, текст и икона; смяна директно от картата; брояч за всеки статус.
- **Обобщение** горе — клик върху брояч филтрира.
- **Търсене** по име, място, район, бележки, описание.
- **Филтри** (панел на цял екран на телефон): място, район, статус, цена, разстояние, капацитет,
  рейтинг, любими, удобства, свободни места, проверени, с бележка, със следващо действие.
- **Сортиране**: препоръчани, цена, близост до плажа, рейтинг, място, статус, любими, последно
  добавени/редактирани, дата на следващо действие.
- **Детайли** със собствен линк, галерия, контакти, история на статусите.
- **Бележки** с автоматично запазване („Запазване…/Запазено"), с кой и кога.
- Добавяне, редакция, **дублиране, архивиране, изтриване с потвърждение и връщане (Undo)**.
- Предупреждение при **невалиден линк** и при **възможен дубликат**.
- Няколко линка и снимки за едно бунгало; Google Maps, обаждане, копиране на телефон, споделяне.
- Следващо действие + дата + предупреждение при просрочка.
- **Импорт/експорт CSV и JSON**, резервно копие, изглед за печат.
- Работи **офлайн** (кешира последните данни), **инсталира се** на телефон (PWA), достъпност и контраст.

## 9. Отстраняване на проблеми

**„Невалиден код или без достъп" при вход.** Кодът трябва да е GitHub fine-grained token за
хранилището `bungalooo` с права **Contents: Read and write** (виж [точка 2](#2-код-за-редактиране)).

**Влизам, но не мога да записвам.** Кодът вероятно е само за четене — създайте нов с
**Contents: Read and write**.

**Промяна на друг не се вижда веднага.** Синхронизацията е чрез опресняване: при редакторите до
~12 секунди; при публичните посетители до няколко минути (заради кеша на GitHub). Издърпайте
страницата надолу за ръчно опресняване.

**Сайтът не се обновява след промяна по кода.** Изпълнихте ли `npm run deploy`? Изчакайте 1–2 мин.

## 10. Структура на проекта

```
Bungalooo/
├─ index.html
├─ vite.config.js          # Vite + PWA; REPO = име на GitHub хранилището (базов път)
├─ scripts/
│  ├─ gen-icons.mjs         # Генерира иконите на приложението
│  └─ create-data-branch.mjs# Създава клона „data" с началния data.json
├─ public/                 # favicon, икони на PWA
└─ src/
   ├─ config.js            # ⭐ Хора, статуси, удобства, райони
   ├─ seed.js              # Началните 23 предложения (за първоначално зареждане)
   ├─ github.js            # „Базата" върху GitHub (четене/запис на data.json)
   ├─ styles.css           # Всички стилове
   ├─ data/store.js        # Слой за данните (опресняване + запис с опашка)
   ├─ context/             # Вход, данни, интерфейс, съобщения
   ├─ hooks/ · utils/       # Помощни функции
   └─ components/          # Навигация, карти, детайли, форма, филтри…
```

Данните живеят в клон **`data`**, файл **`data.json`**. Публикуването на сайта става с
`npm run deploy` (клон `gh-pages`).

## 11. Как е защитено

- **Няма таен код в приложението.** Кодът за редактиране (GitHub token) се въвежда от потребителя
  и се пази само в неговия браузър.
- **Четене — публично; запис — само с валиден код** с права за запис в хранилището.
- **Пълна история**: всяка промяна е commit в клона `data` (може да се върне назад).
- **Валидиране** на данните и **защита от опасни/невалидни линкове** (само `http`, `https`,
  `tel`, `mailto`).
- Потребителският текст се показва **безопасно** (React екранира съдържанието).
- Изтриването изисква **потвърждение** и има **връщане (Undo)**.
