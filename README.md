# Bungalooo 🏖️

Уеб приложение за **сравнение и избор на бунгала по Южното българско Черноморие**.
Няколко души (Ник, Дани, Мари, Иван) разглеждат предложения, добавят бележки,
сменят статуси и виждат актуалната информация от всяко устройство — в реално време.

- **Публичен режим** — всеки с линка разглежда, търси, филтрира, отваря линкове (без промяна).
- **Режим за редактиране** — с обща парола; след вход всеки избира кой е и промените се
  записват с неговото име и цвят.

Интерфейсът е изцяло на български, оптимизиран първо за телефон, работи и офлайн, и може да
се добави на началния екран като приложение (PWA).

## 1. Бърз преглед локално

Ако искате само да видите как изглежда и работи, **без никаква настройка**:

```bash
npm install
npm run dev
```

Отворете адреса, който се показва (обикновено `http://localhost:5173`).

Приложението тръгва в **Демо режим** — данните се пазят само в този браузър (без облак).
Натиснете **Вход**, въведете каквато и да е парола, изберете кой сте и натиснете
**„Зареди началните 23 предложения"**. Така пробвате всичко: добавяне, статуси, бележки,
филтри, сортиране, детайли и т.н.

> За реален сайт, който се синхронизира между устройства и има защитена редакция,
> направете настройката с Firebase по-долу.

## 2. Технологии

| Слой | Избор | Защо |
|------|-------|------|
| Интерфейс | React + Vite | Бързо, леко, лесно за поддръжка |
| База данни | Cloud Firestore | Синхронизация в реално време + работа офлайн |
| Вход | Firebase Authentication | Сигурна проверка на паролата, без таен код в приложението |
| Хостинг | Firebase Hosting | Публикуване с една команда, безплатен план, публичен линк |
| Мобилно | PWA (service worker) | Добавяне на начален екран, работа при слаба връзка |

Всичко е на **един акаунт (Google/Firebase)** и е **безплатно** за този мащаб на ползване.

## 3. Пълна настройка с Firebase

Правите го **веднъж**. Отнема около 10 минути. Нужен е само Google акаунт.

### 3.1. Създайте Firebase проект

1. Отидете на <https://console.firebase.google.com> и влезте с Google акаунт.
2. **Add project** → име (напр. `bungalooo`) → изключете Google Analytics (не е нужен) → **Create project**.

### 3.2. Добавете уеб приложение и вземете ключовете

1. В проекта натиснете иконата **`</>`** (Web).
2. Дайте му псевдоним (напр. `bungalooo-web`) → **Register app**.
3. Ще видите блок `const firebaseConfig = { ... }`. Оттам ви трябват стойностите.
4. В папката на проекта **копирайте** файла `.env.example` като **`.env.local`** и попълнете:

   ```env
   VITE_FIREBASE_API_KEY=...
   VITE_FIREBASE_AUTH_DOMAIN=вашпроект.firebaseapp.com
   VITE_FIREBASE_PROJECT_ID=вашпроект
   VITE_FIREBASE_STORAGE_BUCKET=вашпроект.appspot.com
   VITE_FIREBASE_MESSAGING_SENDER_ID=...
   VITE_FIREBASE_APP_ID=...
   VITE_EDITOR_EMAIL=editor@bungalooo.app
   ```

   > Тези стойности **не са тайни** — те са публични идентификатори. Реалната защита идва
   > от правилата на Firestore и от паролата за вход. Файлът `.env.local` не се качва в Git
   > само за по-голяма чистота.

### 3.3. Създайте базата данни (Firestore)

1. В менюто вляво: **Build → Firestore Database → Create database**.
2. Изберете **Production mode** → регион **eur3 (Europe)** → **Enable**.

### 3.4. Включете входа с парола и създайте редакторския акаунт

1. **Build → Authentication → Get started**.
2. **Sign-in method → Email/Password → Enable → Save**.
3. Раздел **Users → Add user**:
   - **Email:** `editor@bungalooo.app` (същият като `VITE_EDITOR_EMAIL`)
   - **Password:** *това е общата парола за редактиране* — измислете силна и я запишете.
4. Готово. Всеки, който знае тази парола, може да редактира. (Как да я смените — точка 8.3.)

### 3.5. Инсталирайте Firebase CLI и свържете проекта

```bash
npm install -g firebase-tools     # веднъж на компютъра
firebase login                    # влизате със същия Google акаунт
```

Отворете `.firebaserc` и заменете `ЗАМЕНЕТЕ-С-ID-НА-ВАШИЯ-FIREBASE-ПРОЕКТ` с **Project ID**
на вашия проект (виждате го в Project settings). Или изпълнете:

```bash
firebase use --add
```

### 3.6. Публикувайте правилата за достъп

Правилата (`firestore.rules`) казват: **всеки може да чете, само редакторът може да пише**.

```bash
firebase deploy --only firestore:rules
```

> Ако в стъпка 3.4 сте ползвали друг имейл вместо `editor@bungalooo.app`, сменете го и в
> `firestore.rules` (реда с `request.auth.token.email == '...'`), после публикувайте отново правилата.

## 4. Публикуване онлайн

```bash
npm run deploy
```

Тази команда прави production build и качва сайта във Firebase Hosting. Накрая се показва
**публичният линк**, напр.:

```
Hosting URL: https://вашпроект.web.app
```

Това е линкът, който споделяте. 🎉

> `npm run deploy` = `vite build && firebase deploy`. Ако искате да качите само сайта
> (без правилата), ползвайте `npm run deploy:hosting`.

**Първо зареждане на данните:** отворете сайта, натиснете **Вход** с общата парола, изберете кой
сте и натиснете **„Зареди началните 23 предложения"**. Оттам нататък всичко се синхронизира
автоматично между всички устройства.

## 5. Споделяне с други хора

- **Публичният линк** (`https://вашпроект.web.app`) — давате го на когото искате. Те виждат
  всичко само за разглеждане.
- **Общата парола за редактиране** — давате я само на четиримата (Ник, Дани, Мари, Иван).
  След вход всеки избира името си; промените се отбелязват с неговия цвят.
- Всяко бунгало има и **собствен линк** (бутон „Копирай линк" / „Сподели"), който може да
  се прати директно на човек (напр. в семейния чат).

## 6. Обновяване

Когато промените нещо по кода или искате да качите нова версия:

```bash
git pull            # ако ползвате Git (по избор)
npm install         # само ако има нови зависимости
npm run deploy
```

Потребителите получават новата версия автоматично при следващото отваряне (service worker-ът
се обновява сам).

## 7. Резервно копие

Данните са в облака (Firestore) и Google ги пази надеждно, но е добре да правите и собствени копия.

- **Резервно копие:** меню **⋯ → „Изнеси JSON (резервно копие)"**. Сваля се файл с всички данни.
- **Възстановяване:** меню **⋯ → „Внеси / възстанови от файл"** → изберете JSON файла →
  **„Замести всички"**.
- Можете също да изнесете и внесете **CSV** (удобно за Excel / Google Sheets).

Съветваме да сваляте по едно JSON копие от време на време (напр. веднъж седмично по време на
активното планиране).

## 8. Настройки за поддръжка

Всичко важно се сменя на едно място: **`src/config.js`**.

### 8.1. Хора (имена и цветове)

```js
export const USERS = [
  { id: 'nik',  name: 'Ник',  color: '#2563eb' },
  { id: 'dani', name: 'Дани', color: '#0d9488' },
  { id: 'mari', name: 'Мари', color: '#db2777' },
  { id: 'ivan', name: 'Иван', color: '#d97706' },
]
```

Добавете, махнете или преименувайте хора и сменете цветовете. После `npm run deploy`.

### 8.2. Статуси, удобства, райони

В същия файл са и статусите (текст, цвят, икона), списъкът с удобства и предложените райони.

### 8.3. Смяна на общата парола за редактиране

Firebase конзола → **Authentication → Users** → до реда с `editor@bungalooo.app` →
**⋮ → Reset password** (или изтрийте и създайте наново с нова парола). Нищо в кода не се променя.

## 9. Функции

- Карти с предложения: снимка, име, място, цена, разстояние до плажа, рейтинг, капацитет,
  удобства с икони, статус, любимо, кратка бележка.
- **Статуси** с цвят, текст и икона; смяна директно от картата; брояч за всеки статус.
- **Обобщение** горе — клик върху брояч филтрира списъка.
- **Търсене** по име, място, район, бележки, описание.
- **Филтри** (в удобен панел): място, район, статус, цена (мин/макс), разстояние, капацитет,
  рейтинг, любими, удобства, свободни места, проверени, с бележка, със следващо действие.
- **Сортиране**: препоръчани, цена, близост до плажа, рейтинг, място, статус, любими,
  последно добавени/редактирани, дата на следващо действие.
- **Детайли** със собствен URL, галерия със снимки, всички данни, контакти, история на статусите.
- **Бележки** с автоматично запазване („Запазване…" / „Запазено"), с кой и кога.
- Добавяне, редакция, **дублиране, архивиране, изтриване с потвърждение и връщане (Undo)**.
- Проверка на данните и предупреждение при **невалиден линк** и при **възможен дубликат**.
- Няколко линка и няколко снимки за едно бунгало.
- Google Maps, обаждане, копиране на телефон, споделяне, копиране на директен линк.
- Следващо действие + дата + предупреждение при просрочена дата.
- **Импорт/експорт CSV и JSON**, резервно копие, изглед за печат.
- Работи **офлайн**, **инсталира се** на телефон (PWA), достъпност и добър контраст.

## 10. Отстраняване на проблеми

**Показва „Демо" горе вдясно.**
Липсват Firebase ключове — попълнете `.env.local` (точка 3.2) и рестартирайте `npm run dev`
(или направете `npm run deploy`).

**„Грешна парола" при вход.**
Паролата е тази на акаунта `editor@bungalooo.app` от Firebase → Authentication → Users
(точка 3.4). Проверете и че Email/Password провайдърът е включен.

**Виждам данните, но не мога да записвам (в облака).**
Публикувайте правилата: `firebase deploy --only firestore:rules`. Ако сте сменили имейла на
редактора, сменете го и в `firestore.rules`.

**`firebase: command not found`.**
`npm install -g firebase-tools`, после `firebase login`.

**Снимка не се вижда.**
Показва се placeholder „Няма снимка". Проверете, че адресът е директен линк към изображение
(завършва на `.jpg`, `.png` и т.н.) и е публично достъпен.

**Данните изчезнаха след смяна на браузър/устройство (в Демо режим).**
Демо режимът пази данни само локално. За синхронизация направете настройката с Firebase.

```
   ├─ data/store.js        # Слой за данните (Firestore или localStorage)
   ├─ context/             # Вход, данни, интерфейс, съобщения
   ├─ hooks/               # Онлайн статус, инсталиране (PWA)
   ├─ utils/               # Форматиране, валидация, CSV, споделяне и др.
   └─ components/          # Навигация, карти, детайли, форма, филтри и др.
```

## 12. Как е защитено

- **Няма тайна парола в кода.** Паролата за редактиране е паролата на един Firebase акаунт и се
  въвежда във формата за вход. Проверява се сигурно от Firebase.
- **Правила за достъп** (`firestore.rules`): публиката може само да чете; писане е позволено само
  на влезлия редакторски акаунт.
- **Валидиране** на въведените данни и **защита от опасни/невалидни линкове** (разрешени са само
  `http`, `https`, `tel`, `mailto`).
- Потребителският текст се показва **безопасно** (React екранира съдържанието автоматично).
- Изтриването изисква **потвърждение** и има **връщане (Undo)**.
