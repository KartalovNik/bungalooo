# Bungalooo 🏖️

Уеб приложение за **сравнение и избор на бунгала по Южното българско Черноморие**.
Няколко души (Ник, Дани, Мари, Иван) разглеждат предложения, добавят бележки, сменят статуси
и виждат актуалната информация от всяко устройство — в реално време.

- 🌐 **Живо приложение:** <https://kartalovnik.github.io/bungalooo/>
- 💻 **Код (GitHub):** <https://github.com/KartalovNik/bungalooo>

**Как е устроено:**
- **Хостинг → GitHub Pages** (сайтът е публичен, инсталира се на телефона, безплатно).
- **База данни → Firebase Cloud Firestore** (общите данни се синхронизират между всички устройства).

> ⚠️ **В момента сайтът е в „Демо режим".** Работи, но данните се пазят само на всяко
> устройство поотделно. За да заработи **общата синхронизация** между четиримата, трябва да се
> свърже безплатната база Firebase — виж [точка 2](#2-включване-на-общата-база-firebase).

## 1. Как да разглеждам и ползвам

Отворете <https://kartalovnik.github.io/bungalooo/> на телефон или компютър.

- **Публичен режим** (по подразбиране) — всеки разглежда, търси, филтрира, отваря линкове.
- **Режим за редактиране** — натиснете **Вход**, въведете общата парола, изберете кой сте
  (Ник / Дани / Мари / Иван). Всяка ваша промяна се записва с вашето име и цвят.

**Добавяне на приложението на телефона:** отворете сайта в браузъра → менюто на браузъра →
„Добави на началния екран".

## 2. Включване на общата база (Firebase)

Това се прави **веднъж** и включва синхронизацията между всички. Трябва само Google акаунт.
Целият процес е през уебсайта на Firebase — **не е нужна нищо да инсталирате.**

### 2.1. Създайте Firebase проект

1. Влезте в <https://console.firebase.google.com> с Google акаунт.
2. **Add project** → име (напр. `bungalooo`) → изключете Analytics → **Create project**.

### 2.2. Вземете „ключовете" на проекта

1. Натиснете иконата **`</>`** (Web) → псевдоним `bungalooo-web` → **Register app**.
2. Ще видите блок `const firebaseConfig = { apiKey: "...", ... }`.
3. **Изпратете тези стойности на човека, който поддържа кода** (или ги сложете сами — виж
   [точка 7.2](#72-свързване-с-firebase)). Той ще ги добави и ще преиздаде сайта, след което
   „Демо" надписът изчезва и синхронизацията заработва.

> Тези стойности **не са тайни** — те са публични идентификатори. Реалната защита идва от
> правилата по-долу и от паролата за вход.

### 2.3. Създайте базата (Firestore)

**Build → Firestore Database → Create database** → **Production mode** → регион
**eur3 (Europe)** → **Enable**.

### 2.4. Сложете правилата за достъп

Отворете таб **Rules** във Firestore и заменете съдържанието с това (то е и във файла
`firestore.rules`), после **Publish**:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    function isEditor() {
      return request.auth != null
        && request.auth.token.email == 'editor@bungalooo.app';
    }
    match /bungalows/{id} {
      allow read: if true;
      allow create, update: if isEditor()
        && request.resource.data.name is string
        && request.resource.data.name.size() <= 200;
      allow delete: if isEditor();
    }
    match /{document=**} { allow read, write: if false; }
  }
}
```

Правилото значи: **всеки може да чете, само редакторът може да пише.**

### 2.5. Включете входа с парола и създайте редакторския акаунт

1. **Build → Authentication → Get started**.
2. **Sign-in method → Email/Password → Enable → Save**.
3. Таб **Users → Add user**:
   - **Email:** `editor@bungalooo.app`
   - **Password:** *това е общата парола за редактиране* — измислете силна и я запишете.

Готово. Всеки от четиримата, който знае тази парола, влиза в режим за редактиране.

### 2.6. Заредете началните предложения

След като сайтът е свързан с базата (Демо надписът е изчезнал): отворете го → **Вход** с общата
парола → изберете кой сте → **„Зареди началните 23 предложения"**. Оттам нататък всичко се
синхронизира автоматично между всички устройства.

## 3. Публикуване на промени

Сайтът се качва с **една команда** (нужни са Node.js и еднократно `npm install`):

```bash
npm run deploy
```

Това прави build и качва новата версия на GitHub Pages (клон `gh-pages`). След минута-две
промените са живи на <https://kartalovnik.github.io/bungalooo/>. Потребителите получават новата
версия автоматично при следващото отваряне.

## 4. Споделяне с другите

- **Публичният линк** — <https://kartalovnik.github.io/bungalooo/> — давате го на когото искате;
  те виждат всичко само за разглеждане.
- **Общата парола за редактиране** — само за четиримата.
- Всяко бунгало има и **собствен линк** (бутони „Сподели" / „Копирай линк"), който може да се
  прати директно на човек (в семейния чат).

## 5. Настройки за поддръжка

Всичко важно е в **`src/config.js`** (след промяна → `npm run deploy`):

- **Хора (имена и цветове):**

  ```js
  export const USERS = [
    { id: 'nik',  name: 'Ник',  color: '#2563eb' },
    { id: 'dani', name: 'Дани', color: '#0d9488' },
    { id: 'mari', name: 'Мари', color: '#db2777' },
    { id: 'ivan', name: 'Иван', color: '#d97706' },
  ]
  ```

- **Статуси, удобства, предложени райони** — също там.

**Смяна на общата парола:** Firebase конзола → **Authentication → Users** → до реда с
`editor@bungalooo.app` → **⋮ → Reset password**. Нищо в кода не се пипа.

## 6. Резервно копие

- **Сваляне:** меню **⋯ → „Изнеси JSON (резервно копие)"**.
- **Възстановяване:** меню **⋯ → „Внеси / възстанови от файл"** → изберете файла → **„Замести всички"**.
- Поддържа се и **CSV** (за Excel / Google Sheets).

Съветваме да сваляте по едно JSON копие от време на време по време на активното планиране.

## 7. Локална разработка

### 7.1. Стартиране на компютъра

```bash
npm install
npm run dev
```

Без Firebase ключове приложението тръгва в **Демо режим** (данни само локално) — удобно за
пробване.

### 7.2. Свързване с Firebase

Копирайте `.env.example` като **`.env.local`** и попълнете стойностите от
[точка 2.2](#22-вземете-ключовете-на-проекта). За да заработи синхронизацията **и на живия
сайт**, същите стойности се записват във файл **`.env.production`** (публичните ключове на
Firebase може спокойно да стоят в кода) и се прави `npm run deploy`.

> Ако сменяте името на GitHub хранилището, сменете `REPO` в **`vite.config.js`**
> (използва се за адреса на подпапката).

## 8. Функции

- Карти: снимка, име, място, цена, разстояние до плажа, рейтинг, капацитет, удобства с икони,
  статус, любимо, кратка бележка.
- **Статуси** с цвят, текст и икона; смяна директно от картата; брояч за всеки статус.
- **Обобщение** горе — клик върху брояч филтрира.
- **Търсене** по име, място, район, бележки, описание.
- **Филтри** (панел на цял екран на телефон): място, район, статус, цена, разстояние, капацитет,
  рейтинг, любими, удобства, свободни места, проверени, с бележка, със следващо действие.
- **Сортиране**: препоръчани, цена, близост до плажа, рейтинг, място, статус, любими, последно
  добавени/редактирани, дата на следващо действие.
- **Детайли** със собствен линк, галерия, контакти, история на статусите.
- **Бележки** с автоматично запазване („Запазване…/Запазено"), с кой и кога.
- Добавяне, редакция, **дублиране, архивиране, изтриване с потвърждение и връщане (Undo)**.
- Предупреждение при **невалиден линк** и при **възможен дубликат**.
- Няколко линка и снимки за едно бунгало; Google Maps, обаждане, копиране на телефон, споделяне.
- Следващо действие + дата + предупреждение при просрочка.
- **Импорт/експорт CSV и JSON**, резервно копие, изглед за печат.
- Работи **офлайн**, **инсталира се** на телефон (PWA), добра достъпност и контраст.

## 9. Отстраняване на проблеми

**Пише „Демо" горе вдясно.** Базата Firebase още не е свързана — направете
[точка 2](#2-включване-на-общата-база-firebase) и `npm run deploy` (виж 7.2).

**„Грешна парола" при вход.** Паролата е тази на акаунта `editor@bungalooo.app` от Firebase →
Authentication → Users. Проверете и че Email/Password е включен.

**Виждам данните, но не мога да записвам.** Проверете, че сте публикували правилата
([точка 2.4](#24-сложете-правилата-за-достъп)). Ако сте ползвали друг имейл за редактора,
сменете го и в правилата.

**Снимка не се вижда.** Показва се placeholder „Няма снимка". Адресът трябва да е директен
публичен линк към изображение.

**Сайтът не се обновява след промяна.** Изпълнихте ли `npm run deploy`? Изчакайте 1–2 минути и
презаредете (издърпайте страницата надолу за refresh на телефон).

## 10. Структура на проекта

```
Bungalooo/
├─ index.html
├─ vite.config.js          # Vite + PWA; REPO = име на GitHub хранилището (базов път)
├─ firestore.rules         # Правила: всички четат, само редакторът пише
├─ .env.example            # Образец за Firebase ключовете
├─ scripts/gen-icons.mjs   # Генерира иконите на приложението
├─ public/                 # favicon, икони на PWA
└─ src/
   ├─ config.js            # ⭐ Хора, статуси, удобства, райони
   ├─ seed.js              # Началните 23 предложения
   ├─ firebase.js          # Свързване с Firebase (или Демо режим)
   ├─ styles.css           # Всички стилове
   ├─ data/store.js        # Слой за данните (Firestore или локално)
   ├─ context/             # Вход, данни, интерфейс, съобщения
   ├─ hooks/ · utils/       # Помощни функции
   └─ components/          # Навигация, карти, детайли, форма, филтри…
```

Публикуването на GitHub Pages става с `npm run deploy` (пакетът `gh-pages` качва папка `dist`
в клон `gh-pages`; GitHub Pages сервира оттам).

## 11. Как е защитено

- **Няма тайна парола в кода.** Паролата за редактиране е паролата на един Firebase акаунт и се
  въвежда във формата за вход; проверява се сигурно от Firebase.
- **Правила за достъп** (`firestore.rules`): публиката само чете; пише само влезлият редактор.
- **Валидиране** на данните и **защита от опасни/невалидни линкове** (само `http`, `https`,
  `tel`, `mailto`).
- Потребителският текст се показва **безопасно** (React екранира съдържанието).
- Изтриването изисква **потвърждение** и има **връщане (Undo)**.
