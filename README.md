# Bungalooo 🏖️

Уеб приложение за **сравнение и избор на бунгала по Южното българско Черноморие**.
Няколко души (Ник, Дани, Мари, Иван) разглеждат предложения, добавят бележки, сменят статуси
и виждат актуалната информация от всяко устройство — в реално време.

- 🌐 **Живо приложение:** <https://kartalovnik.github.io/bungalooo/>
- 💻 **Код (GitHub):** <https://github.com/KartalovNik/bungalooo>

**Как е устроено:**
- **Хостинг → GitHub Pages** (сайтът е публичен, инсталира се на телефона, безплатно).
- **База данни → Supabase** (общите данни се синхронизират между всички устройства). Влизате
  в Supabase **директно с вашия GitHub акаунт** — без нова регистрация. Firebase НЕ се използва.

> ⚠️ **В момента сайтът е в „Демо режим".** Работи, но данните се пазят само на всяко устройство
> поотделно. За да заработи **общата синхронизация** между четиримата, трябва еднократно да се
> свърже безплатната база — виж [точка 2](#2-включване-на-общата-база-supabase).

## 1. Как да разглеждам и ползвам

Отворете <https://kartalovnik.github.io/bungalooo/> на телефон или компютър.

- **Публичен режим** (по подразбиране) — всеки разглежда, търси, филтрира, отваря линкове.
- **Режим за редактиране** — натиснете **Вход**, въведете общата парола, изберете кой сте
  (Ник / Дани / Мари / Иван). Всяка ваша промяна се записва с вашето име и цвят.

**Добавяне на приложението на телефона:** отворете сайта → менюто на браузъра → „Добави на
началния екран".

## 2. Включване на общата база (Supabase)

Прави се **веднъж**, отнема ~5 минути и е **само в браузъра — нищо за инсталиране.**
Влизате с вашия GitHub акаунт.

### 2.1. Създайте проекта

1. Отворете <https://supabase.com> → **Start your project** → **Continue with GitHub**.
2. **New project** → име `bungalooo` → измислете **Database password** (запишете я някъде) →
   регион **Central EU (Frankfurt)** → **Create new project**. Изчакайте ~2 минути да се създаде.

### 2.2. Създайте таблицата и правилата

1. Вляво изберете **SQL Editor** → **New query**.
2. Отворете файла [`supabase/setup.sql`](supabase/setup.sql) от проекта, копирайте **цялото**
   съдържание, поставете го и натиснете **Run**.
   (Създава таблицата, включва правилата „всеки чете, само редакторът пише" и синхронизацията.)

### 2.3. Създайте редакторския акаунт (= общата парола)

1. Вляво **Authentication → Users → Add user → Create new user**.
2. **Email:** `editor@bungalooo.app`
3. **Password:** *това е общата парола за редактиране* — измислете силна и я запишете.
4. Отметнете **Auto Confirm User** → **Create user**.

### 2.4. Вземете двете стойности за приложението

Вляво **Project Settings → API**. Трябват ви:

- **Project URL** (напр. `https://xxxxx.supabase.co`)
- **API keys → `anon` `public`** (дълъг ключ)

> Тези две стойности **не са тайни** — публични са по дизайн. Сигурността идва от правилата
> (RLS) и от паролата за вход. **Изпратете ги на човека, който поддържа кода**, или ги сложете
> сами (виж [точка 7.2](#72-свързване-с-базата)). Той ги вгражда и преиздава сайта — надписът
> „Демо" изчезва и синхронизацията заработва.

### 2.5. Заредете началните предложения

След свързването: отворете сайта → **Вход** с общата парола → изберете кой сте →
**„Зареди началните 23 предложения"**. Оттам нататък всичко е общо за всички устройства.

## 3. Публикуване на промени

Сайтът се качва с **една команда** (нужни са Node.js и еднократно `npm install`):

```bash
npm run deploy
```

Прави build и качва новата версия на GitHub Pages. След минута-две е живо на
<https://kartalovnik.github.io/bungalooo/>. Потребителите получават новата версия автоматично.

## 4. Споделяне с другите

## 5. Настройки за поддръжка

Всичко важно е в **`src/config.js`** (след промяна → `npm run deploy`):

- **Хора (имена и цветове):**

  ```js
  export const USERS = [
    { id: 'nik',  name: 'Ник',  color: '#2563eb' },
    { id: 'dani', name: 'Дани', color: '#0d9488' },
    { id: 'mari', name: 'Мари', color: '#db2777' },
    { id: 'ivan', name: 'Иван', color: '#d97706' },
  ]
  ```

- **Статуси, удобства, предложени райони** — също там.

**Смяна на общата парола:** Supabase → **Authentication → Users** → до реда с
`editor@bungalooo.app` → **⋮ → Reset password / Update user**.

## 6. Резервно копие

- **Сваляне:** меню **⋯ → „Изнеси JSON (резервно копие)"**.
- **Възстановяване:** меню **⋯ → „Внеси / възстанови от файл"** → изберете файла → **„Замести всички"**.
- Поддържа се и **CSV** (за Excel / Google Sheets).

Съветваме да сваляте по едно JSON копие от време на време по време на активното планиране.

## 7. Локална разработка

### 7.1. Стартиране на компютъра

```bash
npm install
npm run dev
```

Без ключове приложението тръгва в **Демо режим** (данни само локално) — удобно за пробване.

### 7.2. Свързване с базата

Копирайте `.env.example` като **`.env.local`** и попълнете двете стойности от
[точка 2.4](#24-вземете-двете-стойности-за-приложението). За да заработи синхронизацията **и на
живия сайт**, същите стойности се записват във файл **`.env.production`** (публичните ключове на
Supabase може спокойно да стоят в кода) и се прави `npm run deploy`.

> Ако сменяте името на GitHub хранилището, сменете `REPO` в **`vite.config.js`**.

## 8. Функции

- Карти: снимка, име, място, цена, разстояние до плажа, рейтинг, капацитет, удобства с икони,
  статус, любимо, кратка бележка.
- **Статуси** с цвят, текст и икона; смяна директно от картата; брояч за всеки статус.
- **Обобщение** горе — клик върху брояч филтрира.
- **Търсене** по име, място, район, бележки, описание.
- **Филтри** (панел на цял екран на телефон): място, район, статус, цена, разстояние, капацитет,
  рейтинг, любими, удобства, свободни места, проверени, с бележка, със следващо действие.
- **Сортиране**: препоръчани, цена, близост до плажа, рейтинг, място, статус, любими, последно
  добавени/редактирани, дата на следващо действие.
- **Детайли** със собствен линк, галерия, контакти, история на статусите.
- **Бележки** с автоматично запазване („Запазване…/Запазено"), с кой и кога.
- Добавяне, редакция, **дублиране, архивиране, изтриване с потвърждение и връщане (Undo)**.
- Предупреждение при **невалиден линк** и при **възможен дубликат**.
- Няколко линка и снимки за едно бунгало; Google Maps, обаждане, копиране на телефон, споделяне.
- Следващо действие + дата + предупреждение при просрочка.
- **Импорт/експорт CSV и JSON**, резервно копие, изглед за печат.
- Работи **офлайн** (кешира последните данни), **инсталира се** на телефон (PWA), достъпност и контраст.

## 9. Отстраняване на проблеми

**Пише „Демо" горе вдясно.** Базата още не е свързана — направете
[точка 2](#2-включване-на-общата-база-supabase) и `npm run deploy` (виж 7.2).

**„Грешна парола" при вход.** Паролата е тази на акаунта `editor@bungalooo.app` от Supabase →
Authentication → Users. Проверете, че сте отметнали **Auto Confirm** при създаването му.

**Виждам данните, но не мога да записвам.** Изпълнихте ли SQL-а от
[точка 2.2](#22-създайте-таблицата-и-правилата)? Той включва правилата за писане.

**Сайтът „заспива" / бавен е при първо отваряне след дни.** Безплатните Supabase проекти се
поставят в пауза при ~7 дни без активност. Влезте в Supabase конзолата и натиснете **Restore**,
или просто изчакайте няколко секунди — събужда се при първа заявка.

**Сайтът не се обновява след промяна.** Изпълнихте ли `npm run deploy`? Изчакайте 1–2 минути и
презаредете.

## 10. Структура на проекта

```
Bungalooo/
├─ index.html
├─ vite.config.js          # Vite + PWA; REPO = име на GitHub хранилището (базов път)
├─ supabase/setup.sql      # SQL за таблицата + правилата (поставя се в Supabase)
├─ .env.example            # Образец за двете стойности на Supabase
├─ scripts/gen-icons.mjs   # Генерира иконите на приложението
├─ public/                 # favicon, икони на PWA
└─ src/
   ├─ config.js            # ⭐ Хора, статуси, удобства, райони
   ├─ seed.js              # Началните 23 предложения
   ├─ backend.js           # Свързване със Supabase (или Демо режим)
   ├─ styles.css           # Всички стилове
   ├─ data/store.js        # Слой за данните (Supabase или локално)
   ├─ context/             # Вход, данни, интерфейс, съобщения
   ├─ hooks/ · utils/       # Помощни функции
   └─ components/          # Навигация, карти, детайли, форма, филтри…
```

Публикуването на GitHub Pages става с `npm run deploy` (пакетът `gh-pages` качва папка `dist`
в клон `gh-pages`).

## 11. Как е защитено

- **Няма тайна парола в кода.** Паролата за редактиране е паролата на един акаунт в базата и се
  въвежда във формата за вход; проверява се сигурно от Supabase.
- **Правила за достъп (RLS)**: публиката само чете; пише само влезлият редактор.
- Двете стойности в кода (URL + `anon` ключ) са **публични идентификатори**, не тайни.
- **Валидиране** на данните и **защита от опасни/невалидни линкове** (само `http`, `https`,
  `tel`, `mailto`).
- Потребителският текст се показва **безопасно** (React екранира съдържанието).
- Изтриването изисква **потвърждение** и има **връщане (Undo)**.
